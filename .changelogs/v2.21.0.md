# Release v2.21.0

Released: 2026-06-26

## Highlights

- **Quieter, more diagnostic logs.** The gateway caches the read-only fill ledger and no longer reprints "Loaded N fills" on every poll, and per-cycle stuck-state messages (no-TP, dust-waiting, low-balance pause) are throttled to actual state transitions instead of repeating every loop â€” so the logs surface real events instead of burying them in noise.
- **Crash-safe config reloads.** A malformed or partially-written config file can no longer crash the gateway or engines â€” the loader falls back to the last-good config and warns once per episode instead of throwing on every read.
- **Automatic dust cleanup.** Stranded sub-minimum "dust" bodies that can't get their own take-profit order are now safely consolidated into the nearest body, under a lock that prevents double-counting against operator or reconcile actions.
- **Resilient Gemini API usage.** Gemini private REST calls are throttled to stay under the rate limit so a reconcile burst can't 429-starve the heartbeat that keeps resting orders alive; transient 429s are retried with backoff, while order placement fails fast (no retry) to rule out a double-place.
- **Accounting correctness.** A sub-cent orphan buy whose cost rounds to zero no longer produces a `NaN` cost basis when merged into a body.

## Added

- Safely auto-consolidate stranded sub-minimum dust bodies into the nearest body, gated by a merge lock (#189)

## Fixed

### Logging & observability
- Cache the read-only fill ledger in the gateway and quiet per-poll reload logs (#183); don't cache the ledger for nonexistent funds (#183 review)
- Throttle per-cycle stuck-state logs (no-TP, dust-waiting, low-balance pause) to state transitions (#187); reset the low-balance pause throttle on cycle reset (#187 review)

### Reliability
- Guard `loadRawConfig` JSON.parse so a bad or partial read can't crash the gateway/engines â€” fall back to last-good config (#185); throttle the reload-failed warning to once per episode (#185 review)

### Orders, cycles & accounting
- Hold `reconcileInProgress` until all dispatched body-TP chains settle, so a fire-and-forget chain can't outlive the lock (#189 review)
- Fix `mergeIntoBody` (and its twin `createNewBody`) producing a `NaN` cost basis on a sub-cent buy whose cost rounds to zero â€” use `??` not `||` so a legitimate zero cost doesn't fall through to the fallback (#191)

### Exchange adapters
- Throttle Gemini private REST to ~5 req/s and retry the heartbeat on 429 so a reconcile burst can't starve it (#193); exclude order placement from 429 retry to avoid a double-place (#194 review)
