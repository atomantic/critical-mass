# Release v2.19.4

Released: 2026-06-24

## Highlights

- **Crash-safe order accounting.** A buy is now persisted before its sell is placed, and `awaiting_sell` rows are recovered after a crash between the two â€” so a failed or interrupted sell can no longer lose a position.
- **More accurate P&L.** Held-cost basis is derived from actually-filled sells rather than placement-time stamps, the fill-ledger/closed-trades cluster of edge cases is resolved, and dry-run now folds entry fees into cost basis for true round-trip fee parity.
- **Steadier engine.** Cycle-pair truth recalculation, reconcile state preservation, an entry-race lock, partial-fill detection, buy de-duplication, and WS-cancelled-partial handling close a series of races and miscounts in the regime engine.
- **Exchange data completeness.** Gemini fills now page correctly per-order and stitch multi-window trade history so orders with more than 500 fills are fully reachable; a shared heartbeat is ref-counted to avoid premature teardown.
- **Resilient connectivity.** A WebSocket pong-timeout watchdog terminates silently-dead connections and reconnects, and the market-data pipeline's dead-metrics and zero-price paths are repaired with multi-pair keying and atomic config writes.
- **Security.** Fund-config routes no longer leak the Telegram bot token or other global secrets.
- **Sharper UpDown signals.** Exits surface inside the no-trade zone, contract outcomes are counted once, and backtests drop a lookahead bias in higher-timeframe seeding.
- **Dashboard fixes.** Prorated Filled-Orders P&L, live-fill refresh, and a fund-switch race in the admin UI are addressed.
- **Operability & open-source prep.** The `critical-mass-ui` PM2 app no longer crash-loops, the README dashboard screenshot now renders, API keys are documented as UI-configurable, and the repo ships safely out of the box from `config.example.json` (no operator config tracked).
- **Dependencies.** Routine security and maintenance bumps across `ws`, `engine.io`/`socket.io`, `react-router`, `qs`, `shell-quote`, `fast-xml-parser`, `brace-expansion`, `esbuild`/`vite`, and `concurrently`.

## Fixed

### Engine & P&L
- Recalculate cycle-pair truth with reconcile state preservation, an entry-race lock, and containment of polled-fill rejections (#121)
- Partial-fill detection, buy de-duplication, WS-cancelled partials, TP-guard, and ladder-count fixes (#122)
- Derive `heldOpenBuyCostBasis` from filled sells, not placement-time stamps (#118)
- Fill-ledger / closed-trades cluster, held-cost, and recalc-preview fixes (#108, #128, #131, #132, #135)
- Epsilon increment flooring plus dry-run fee / forceUpdate parity in adapters (#125)
- Fold entry fee into body cost basis for round-trip fee parity in dry-run (#133, #138)
- `start()` reentrancy guard, dry-run sizeOptimizer persistence, and `normalizeExchangeBlock` copy (#127)

### Order safety & crash recovery
- Persist the buy before sell placement so a failed sell can't lose it (#120)
- Recover `awaiting_sell` rows after a crash between buy-save and sell placement (#129, #136)
- Stop insufficient-funds entry spam and gate entries on real balance (Coinbase)

### Exchange adapters
- Per-order symbol resolution, paginated fills lookup, and ref-counted shared heartbeat (Gemini) (#119)
- Stitch multi-slice `mytrades` windows so orders with more than 500 fills are reachable (Gemini) (#130, #137)

### Market data & connectivity
- Repair the dead metrics pipeline, a nonexistent detector method, and wrong `calculateAllMetrics` args (#117)
- Zero-price and WS-side guards, multi-pair keying, service-start race, `log.warn`, stale-orders, and atomic config write (#123)
- Add a pong-timeout watchdog so dead connections terminate and reconnect (#114)

### UpDown signals
- Surface exits in `NO_TRADE_ZONE`, count contract outcome once, and fix dedup / JSONL / vol-decay (#124)
- Remove lookahead bias from higher-timeframe seeding in the backtest (#115)

### Dashboard / admin UI
- Prorated Filled Orders P&L, error handling, live-fill refresh, fund-switch race, and number inputs (#126)

### Security
- Stop leaking the Telegram bot token and global secrets via fund config routes (#116)

### Operations
- Run `critical-mass-ui` via `node` + `vite.js` with array args to stop the `npx` crash-loop

### Documentation & open-source prep
- Commit the dashboard screenshot that a blanket `*.png` ignore rule kept out of the repo (fixed the broken README image)
- Note that API keys can be configured in the admin UI
- Untrack `config.json` and bootstrap from `config.example.json` so a fresh clone runs with safe dry-run defaults
